## 可复用类模板规范（优化版）

### 一、总体目标
构建**可直接复制使用、接口统一、低耦合、易维护**的算法类模板集合，适用于竞赛与工程场景。

---

### 二、命名规范
1. **类名**：大驼峰命名法（PascalCase）。
2. **方法 / 变量**：全小写 + 下划线（snake_case）。
3. **内部方法**：统一加前缀 `_`。
4. 保持整体风格一致，避免混用命名体系。

---

### 三、接口与使用规范
1. **默认索引**：采用 `1-base`，如使用 `0-base` 必须显式注释说明。
2. **错误返回值**：统一返回 `-1`（如无特殊说明）。
3. **多测支持**：必须提供 `reset()` 方法用于状态清空。
4. **接口设计**：
   * 保持调用方式统一
   * 避免同功能多种重载写法
   * 参数语义清晰、最小化

---

### 四、注释规范
1. 统一使用 `//`，禁止 `/** */`。
2. **类定义前必须说明**：
   * 模板参数类型的必要约束（如是否需要 `operator<`）
   * 使用流程（如有调用顺序）
   * 整体时间复杂度
1. **函数注释要求**：
   * 简要说明功能
   * 标明返回值含义
   * 标注时间复杂度
4. 对非默认行为（如 `0-base`）必须明确说明。

---

### 五、内部实现约束
1. 优先使用 **C风格数组**（性能优先）。
2. 对于空间复杂度为 $O(N \cdot M)$：
   * 若无跨组依赖，建议由调用方分配
   * 或使用 `vector<int> g[N]` 等结构拆分存储
3. 避免隐式动态扩容带来的不确定性能。

---

### 六、模板参数规范
1. 支持模板参数配置：
   ```cpp
   N   // 数据规模
   M   // 辅助规模（如边数）
   MOD // 模数
   INF // 无穷大
   ```
2. 默认值：
   * `N = 1e5 + 9`（或 `N * 17 + 9` 用于 $O(n\log n)$）
   * `M = 1e5 + 9`（或 `2e5 + 9` 用于表示边数）
   * `MOD = 998244353`
   * `INF = 3e18`

---

### 七、泛型与约束
1. 推荐使用模板泛型提高复用性。
2. 必须**显式说明类型要求**（如：
   * 可比较：`operator<`
   * 可拷贝构造
   * 可默认构造）
1. 使用统一检测宏进行编译期约束：
```cpp
// 1 基础检测框架（全局仅定义一次）
#ifndef DEFINE_CHECKER
#define DEFINE_CHECKER(name, ...) \
template<typename U> \
struct name { \
    template<typename V> \
    static char test(decltype((void)(__VA_ARGS__), 0)*); \
    template<typename V> \
    static int test(...); \
    enum { value = sizeof(test<U>(0)) == 1 }; \
    typedef char ASSERT[value ? 1 : -1]; \
};
#endif

// 2 示例：排序器类型检查
DEFINE_CHECKER(CheckSorter,
    V(),              // 默认构造
    V(*(V*)0),        // 拷贝构造
    *(V*)0 < *(V*)0   // 小于比较
)

// 3 使用方式
template<class T>
struct Sorter {
	// 若在这一行报错：类型 T 未实现所需运算符！
    typedef typename CheckSorter<T>::ASSERT _check;
    // ...
};
```
---

### 八、实例化要求
1. 每个结构体定义后需提供一个**示例实例化**：
   * 展示模板参数填写方式
   * 便于直接使用与测试

---

### 九、复杂度标注规范
* **类级别**：标明整体复杂度（如建图 + 查询）
* **函数级别**：标明单次调用复杂度
* 必须准确，不允许模糊描述

---

### 十、设计原则（补充）
* 接口优先：调用体验 > 实现细节
* 稳定优先：避免未定义行为依赖
* 简洁优先：避免过度抽象
* 可维护优先：结构清晰、职责单一

---

该规范适用于竞赛模板库、工程算法库的统一约束，可在保证性能的前提下提升可读性与复用性。



以下是一个标准例子：
```cpp
// 离散化er：用于离散化数据
// 类型 T 需要：
// 默认构造 / 拷贝构造 / < / ==
//
// 使用流程：
// add -> build -> rank/get/inv_get
//
// 整体复杂度：
// build: O(n log n)
// 单次查询: O(log n)

#ifndef DEFINE_CHECKER
#define DEFINE_CHECKER(name, ...) \
template<typename U> \
struct name { \
    template<typename V> \
    static char test(decltype((void)(__VA_ARGS__), 0)*); \
    template<typename V> \
    static int test(...); \
    enum { value = sizeof(test<U>(0)) == 1 }; \
    typedef char ASSERT[value ? 1 : -1]; \
};
#endif

DEFINE_CHECKER(CheckDiser,
    V(),              // 默认构造
    V(*(V*)0),        // 拷贝构造
    *(V*)0 < *(V*)0,  // <
    *(V*)0 == *(V*)0  // ==
)

template<class T, int N = 100000 + 9>
struct Diser {
	// 若在这一行报错：类型 T 未实现所需运算符！
    typedef typename CheckDiser<T>::ASSERT _check;

    int n;
    T a[N];

    Diser() { reset(); }

    // 重置容器，用于多测试用例
    // O(1)
    void reset() {
        n = 0;
    }

    // 添加待离散化元素
    // 若超出容量返回 -1
    // O(1)
    int add(const T& x) {
        if (n >= N) return -1;
        a[n++] = x;
        return 0;
    }

    // 构建离散化表，必须在查询前调用
    // O(n log n)
    void build() {
        std::sort(a, a + n);
        n = std::unique(a, a + n) - a;
    }

    // 返回 x 的排名（1-based）
    // 若不存在，返回其应插入位置（仍为1-based）
    // O(log n)
    int rank(const T& x) const {
        return std::lower_bound(a, a + n, x) - a + 1;
    }

    // 返回 x 离散化后的索引（1-based）
    // 若不存在，返回 -pos（pos为插入位置1-based）
    // O(log n)
    int get(const T& x) const {
        auto it = std::lower_bound(a, a + n, x);
        int pos = (int)(it - a + 1);
        if (it == a + n || !(*it == x)) {
            return -pos;
        }
        return pos;
    }

    // 根据离散化索引（1-based）还原原始值
    // 若非法返回默认值 T{}
    // O(1)
    T inv_get(int idx) const {
        if (idx <= 0 || idx > n) return T{};
        return a[idx - 1];
    }

    // 返回去重后的元素个数
    // O(1)
    int size() const {
        return n;
    }
};

// 示例实例化
Diser<int, 100000 + 9> diser;
```
