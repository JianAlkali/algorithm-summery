[[_动态规划]]
$O\left(n \log n\right)$。有`n`月，你每月结束后可以获得`x`元，每月都有一次花费$a_{i}$元购买物品的机会。求最大购买物品数。这并没有足够典型，但我认为可以放在这。
按时间顺序贪心的可行性：我们的约束前缀和`xtop`一直在变大。若我们已经选了最优的`k`个物品，新来一个物品，可选$→$选一定最优；不可选，若已选物品中有更贵的，将原有物品中最贵的替换为新物品$→$一定不会破坏约束，且一定是选`k`个物品的组合中最优的。
```cpp
int a[N];
void solve(){
    int n,x;
    cin>>n>>x;
    for(int i=0;i<n;i++)
        cin>>a[i];
    priority_queue<int> pq;
    int ans=0, xnow=0, xtop=0;
    for(int i=0;i<n;i++){
        if(xnow+a[i]<=xtop){
            pq.emplace(a[i]);
            xnow+=a[i];
            ans=max(ans,(ll)pq.size());
        }else if(!pq.empty() && a[i]<pq.top()){
            xnow-=pq.top(); xnow+=a[i];
            pq.pop();
            pq.emplace(a[i]);
        }
        xtop+=x;
    }
    cout<<ans<<'\n';
}
```
