[[图论]]

匈牙利算法（Kuhn），通过不断找增广边来达到最大匹配，复杂度$O\left(nm\right)$。下代码例省略了链式前向星的部分。使用最大流Dinic求解二分图最大匹配可优化至$O\left(\sqrt{n}⋅m\right)$。下面是匈牙利算法：
```cpp
int vist[N]; // 仅左部点使用，表示该轮是否vis过
int match[N]; // 仅右部点使用，表示已匹配的左部点
bool dfs(int u,int t){
    if(vist[u]==t) return false;
    vist[u]=t; // 使用时间戳式vis使得不用每次清空vis
    for(int i=head[u];i;i=e[i].nxt){
        int v=e[i].v;
        if(!match[v] || dfs(match[v],t)){ // 寻找增广路
            match[v]=u;
            return true;
        }
    }
    return false;
}
void solve(){
    int n,m,e;
    cin>>n>>m>>e;
    fill(head,head+n+m+1,0);
    fill(vist,vist+n+m+1,0);
    fill(match,match+n+m+1,0);
    ec=0;
    for(int i=0,u,v;i<e;i++){
        cin>>u>>v;
        add(u,v); // 仅需存单边图
}
    int ans=0;
    for(int u=1;u<=n;u++)
        ans+=dfs(u,u);
    cout<<ans<<'\n';
}
```
